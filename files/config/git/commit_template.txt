
# # Conventional commit
# see: https://github.com/conventional-changelog/commitlint/tree/master/%40commitlint/config-conventional
# If you are in nvim, you can use snippets (completion menu will be opened with <c-space>)
#
# `<type>[optional scope]: <description>`
# type        ... e.g. build, chore, ci, docs, feat, fix, style, refactor, revert, perf, test, etc.
# scope       ... e.g. file name, module name, etc.
# description ... title of the commit
#
# ## Types and related gitmojis (about gitmoji: https://gitmoji.dev)
# * build  ......  Changes that affect the build system or external dependencies
#                      🔧 :wrench:                    Add or update configuration files
#                      📦️ :package:                   Add or update compiled files or packages
# * chore  ......  Chores
#                      🔖 :bookmark:                  Release / Version tags
#                      🚀 :rocket:                    Deploy stuff
#                      ⬆️ :arrow_up:                  Upgrade dependencies
#                      ⬇️ :arrow_down:                Downgrade dependencies
#                      📌 :pushpin:                   Pin dependencies to specific versions
#                      🙈 :see_no_evil:               Add or update a .gitignore file
#                      🧑‍💻 :technologist:              Improve developer experience
# * ci  .........  Changes to our CI configuration files and script
#                      💚 :green_heart:               Fix CI Build
#                      👷 :construction_worker:       Add or update CI build system
# * docs  .......  Documentation only changes
#                      📝 :memo:                      Add or update documentation
#                      📄 :page_facing_up:            Add or update license
# * feat  .......  A new feature
#                      ✨ :sparkles: Introduce new features
#                      💄 :lipstick:                  Add or update the UI and style files
#                      🍱 :bento:                     Add or update assets
# * fix  ........  A bug fix
#                      🐛 :bug:                       Fix a bug
#                      🚑️ :ambulance:                 Critical hotfix
#                      🔒️ :lock:                      Fix security or privacy issues
#                      🚨 :rotating_light:            Fix compiler / linter warnings
#                      ✏️ :pencil2:                   Fix typos
#                      🥅 :goal_net:                  Catch errors
#                      🩹 :adhesive_bandage:          Simple fix for a non-critical issue
# * style  ......  Changes that do not affect the meaning of the code
#                      🎨 :art:                       Improve structure / format of the code
#                      🚚 :truck:                     Move or rename resources (e.g.: files, paths, routes)
#                      💡 :bulb:                      Add or update comments in source code
# * refactor  ...  A code change that neither fixes a bug nor adds a feature
#                      🔥 :fire:                      Remove code or files
#                      ♻️ :recycle:                   Refactor code
#                      🏗️ :building_construction:     Make architectural changes
#                      🗑️ :wastebasket:               Deprecate code that needs to be cleaned up
#                      ⚰️ :coffin:                    Remove dead code
# * revert  .....  Revert changes
#                      ⏪️ :rewind:                    Revert changes
# * perf  .......  A code change that improves performance
#                      ⚡️ :zap:                       Improve performance
# * test  .......  Adding missing tests or correcting existing tests
#                      ✅ :white_check_mark:          Add, update, or pass tests
# * misc  .......  Miscellaneous
#                      🎉 :tada:                      Begin a project
#                      🚧 :construction:              Work in progress
#                      🔀 :twisted_rightwards_arrows: Merge branches
